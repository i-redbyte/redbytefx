# v0.3 Tooling (sample)

**Goal:** the `:sample` app is the primary adoption surface: every important demo shows **preview**, **live controls**, **DSL snippet**, and **generated AGSL**, with a deliberate layout on phones, tablets, and wide screens.

## What ships in the sample

- **Catalog:** search, sections, canonical / start-here signals, focus tags, related demos (`Demo.kt`, `HomeScreen.kt`).
- **Demo screen:** shared **`DemoLayout`**: preview panel, controls panel, metadata card with inspection flow, prev/next, follow-ups (`DemoComponents.kt`).
- **Breakpoints:** compact phone (&lt; ~420dp) keeps preview + controls first; **≥ 840dp** uses a two-column **preview | controls** row so tooling reads well on tablets and desktops.
- **Code panels:** expandable DSL and AGSL blocks with **COPY** (clipboard) and **SHARE** (system share sheet). On narrow phones, **COMPARE CODE** opens a full-screen dialog with the same panels.
- **Debug checklist:** per-demo steps aligned with path signal (start-here / canonical / exploratory / foundation).
