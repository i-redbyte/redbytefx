# v0.5 Runtime quality

**Goal:** predictable runtime and sample behavior — performance, lifecycle, and regression confidence without premature micro-optimization.

## What is “done” for this documentation milestone

The following are **in tree** and treated as the **first complete v0.5 documentation + measurement baseline** (not a performance guarantee):

- **[runtime-audit-v0.5.md](runtime-audit-v0.5.md)** — controller/`FxInstance` behavior, representative demos, emulator vs physical device baselines.
- **[runtime-measurement-scenario.md](runtime-measurement-scenario.md)** — repeatable gfxinfo/meminfo checklist and invalidation notes.
- **`FxInstance` setters return `Boolean`**; **`FxController`** invalidates from instance return values; measurement script and `START_DEMO` launch path for `Radar` / `Circuit`.

## What remains incremental (not a reopened milestone)

- Broader device classes and resize/split-screen passes when the runtime architecture changes.
- Optional screenshot/smoke tests for a **small** demo set (infrastructure-dependent).
- Any change driven by **new** platform constraints — document in the audit notes, not as a blanket “v0.5 unfinished”.
