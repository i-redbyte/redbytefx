# v0.4 Authoring UX (guide hub)

**Goal:** writing RedByteFX feels teachable and debuggable for authors coming from raw AGSL or Shadertoy-style shaders.

This hub links the **first complete v0.4 documentation pass** in-tree. It is still not a publication contract; the library remains pre-publication.

## What this milestone added (author-facing)

- **Compiler/runtime messages:** clearer text when a uniform `FxParam` does not belong to the bound effect; hints when raw Kotlin values (ints, strings, booleans) reach the shader emitter instead of DSL expressions.
- **`fn` return-type validation** (see [CHANGELOG.md](../CHANGELOG.md)): body type must match declared `FxValueType` before codegen.
- **Docs:** expanded AGSL comparison, compact **porting checklist**, cookbook promoted from “draft” to maintained companion, cross-links from README.
- **KDoc:** `sample(...)` vs stdlib `sampleUv(...)` called out in core/stdlib/Compose where authors look first.

## Explicitly out of scope for this pass

- Structured / IDE-style diagnostics API (exceptions remain the transport).
- Source maps or line-level shader↔Kotlin mapping.
- IDE features inside `:sample` (it stays a demo, not an IDE).

Future incremental work can deepen the cookbook and tighten naming across `stdlib` under **v0.6 Library shaping**, without reopening v0.4 as a blank slate.
